    1. Create Maven project `prosperity-api` with Spring Boot 3.5.x parent. Dependencies:
       - spring-boot-starter-web, spring-boot-starter-security, spring-boot-starter-data-jpa, spring-boot-starter-validation
       - liquibase-core, postgresql (runtime), jjwt-api + jjwt-impl + jjwt-jackson (0.12.6), java-uuid-generator (5.1.0)
       - spring-boot-starter-test + testcontainers-postgresql (test scope)
       - Java 21, enable virtual threads

    2. Create `ProsperityApplication.java` main class in `fr.kalifazzia.prosperity`.

    3. Create `application.yml`:
       - Datasource: `jdbc:postgresql://${DB_HOST:localhost}:${DB_PORT:5432}/${DB_NAME:prosperity}`
       - JPA: hibernate ddl-auto=validate, show-sql=false
       - Liquibase: enabled, changelog=db/changelog/db.changelog-master.yaml
       - Virtual threads enabled: `spring.threads.virtual.enabled=true`
       - JWT config: `app.jwt.secret=${JWT_SECRET}`, `app.jwt.access-expiry=900` (15min), `app.jwt.refresh-expiry=2592000` (30 days)
       - Server port 8080

    4. Create `application-dev.yml` with debug logging for SQL and security.

    5. Shared domain kernel:
       - `Money.java`: Value object wrapping BigDecimal with RoundingMode.HALF_EVEN, scale 4. Factory method `of(BigDecimal)`, `of(String)`. Implements `add`, `subtract`, `negate`. Immutable record or class with equals/hashCode based on value.
       - `UserId.java`: Simple value object wrapping UUID. Factory method `generate()` using JUG UUIDv7 (`UuidCreator.getTimeOrderedEpoch()`). `of(UUID)` for existing IDs.
       - `AccountId.java`: Same pattern as UserId.

    6. `BaseEntity.java`: MappedSuperclass with `@Id UUID id`, `@CreatedDate Instant createdAt`, `@LastModifiedDate Instant updatedAt`, `@Version Long version`. Enable JPA auditing in a config class.

    7. `JacksonConfig.java`: Configure ObjectMapper for Java 8 time module, fail on unknown properties false.

    8. `GlobalExceptionHandler.java`: @RestControllerAdvice handling MethodArgumentNotValidException (400), AccessDeniedException (403), generic Exception (500). Return consistent error DTO: `{ error: String, details: Map<String,String> }`.

    9. `SecurityConfig.java`:
       - BCryptPasswordEncoder bean with strength 12
       - SecurityFilterChain: disable CSRF (stateless API), enable CORS for SvelteKit origin, stateless session management
       - Headers: CSP ("default-src 'self'"), HSTS (maxAge 31536000, includeSubDomains), X-Frame-Options DENY, X-Content-Type-Options nosniff
       - Permit all on `/api/setup/**`, `/api/auth/**`, `/actuator/health`
       - All other endpoints require authentication (JWT filter will be added in Plan 02)

    10. Liquibase migrations (YAML format, IDs: YYYYMMDD-NN-description):
        - `users` table: id (uuid PK), email (varchar 255, unique, not null), password_hash (varchar 255, not null), display_name (varchar 255, not null), system_role (varchar 50, not null, default 'STANDARD'), preferences (jsonb, default '{}'), force_password_change (boolean, default false), created_at (timestamp not null), updated_at (timestamp not null), version (bigint default 0)
        - `accounts` table: id (uuid PK), name (varchar 255, not null), bank_name (varchar 255), account_type (varchar 50, not null), owner_id (uuid FK to users, not null), currency (varchar 3, default 'EUR'), initial_balance (decimal 19,4 default 0), current_balance (decimal 19,4 default 0), color (varchar 7), created_at, updated_at, version
        - `account_permissions` table: id (uuid PK), account_id (uuid FK), user_id (uuid FK), permission_level (varchar 50, not null), unique constraint on (account_id, user_id)
        - `categories` table: id (uuid PK), name_key (varchar 100, not null), icon (varchar 50), is_default (boolean default false), created_by (uuid FK to users, nullable -- null for defaults), created_at, updated_at
        - `refresh_tokens` table: id (uuid PK), user_id (uuid FK to users, not null), token_hash (varchar 255, not null, unique), expires_at (timestamp not null), created_at (timestamp not null). Add index on token_hash.
        - Seed default categories: Alimentation, Logement, Transport, Sante, Assurance, Loisirs, Restaurants, Shopping, Abonnements, Sport, Epargne, Revenus, Remboursement, Cadeaux, Education, Divers. Set is_default=true, created_by=null.

    11. `Dockerfile`: Multi-stage build. Stage 1: Maven build with `mvn package -DskipTests`. Stage 2: Eclipse Temurin 21 JRE slim, copy JAR, expose 8080, entrypoint java -jar.

    2. Install dependencies:
       - `npm install -D @tailwindcss/vite tailwindcss` (Tailwind CSS 4 Vite plugin)
       - `npm install @inlang/paraglide-js` (Paraglide i18n)
       - `npm install -D @sveltejs/adapter-node` (for Docker deployment)

    3. `vite.config.ts`: Configure with tailwindcss() Vite plugin, paraglide({ project: './project.inlang', outdir: './src/lib/i18n' }), and sveltekit() plugins in that order.

    4. `svelte.config.js`: Use adapter-node.

    5. `app.html`: Standard SvelteKit template with FOUC prevention inline script (reads `theme` cookie, applies `dark` class to html before paint). Set `lang="fr"` default. Include `%sveltekit.head%` and `%sveltekit.body%`.

    6. `app.css`: Import Tailwind CSS 4 with `@import "tailwindcss"`. Add `@theme` block for custom colors (primary, accent). Configure dark mode via `@variant dark (&:where(.dark, .dark *))`.

    7. `app.d.ts`: Extend `App.Locals` with `accessToken?: string` and `user?: { id: string, email: string, role: string, displayName: string }`.

    8. `hooks.server.ts`: Implement BFF auth flow skeleton:
       - `tokenRefresh` handle: if no access_token but refresh_token exists, call `${API_URL}/api/auth/refresh` to get new tokens, set httpOnly cookies (access: 15min, refresh: 30 days, secure, sameSite lax).
       - `authGuard` handle: if no access_token and path is not public (/login, /setup), redirect 303 to /login.
       - If access_token exists, decode JWT payload (base64 middle segment, no verification -- verification is API's job) to populate `event.locals.user`.
       - Export `handle = sequence(tokenRefresh, authGuard)`.
       - Define `API_URL` from env var `API_URL` defaulting to `http://localhost:8080`.

    9. `src/lib/api/client.ts`: Create `apiClient(accessToken: string)` factory returning object with `get(path)`, `post(path, body)`, `put(path, body)`, `patch(path, body)`, `delete(path)`. Each method calls `fetch(API_URL + path, ...)` with Authorization Bearer header and Content-Type application/json. Return raw Response.

    10. `src/lib/stores/preferences.svelte.ts`: Create PreferencesStore class using Svelte 5 runes ($state for theme, locale, defaultCurrency; $derived for resolvedTheme). Include `setTheme()` method that toggles `dark` class on documentElement. Export singleton instance.

    11. Paraglide setup:
        - `project.inlang/settings.json`: sourceLanguageTag "fr", languageTags ["fr", "en"], modules for paraglide.
        - `messages/fr.json`: Core UI strings in French (login, setup, settings, accounts, common actions, session_expired).
        - `messages/en.json`: Same keys in English.

    12. `src/routes/+layout.svelte`: Root layout -- minimal for now, just `<slot />` with basic html structure. Apply Tailwind base styles. Import app.css.

    13. `Dockerfile`: Multi-stage. Stage 1: Node 22 alpine, npm ci, npm run build. Stage 2: Node 22 alpine slim, copy build output, expose 3000, run with `node build`.
