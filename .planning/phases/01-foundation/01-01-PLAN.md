  <action>
    1. Create Maven project `prosperity-api` with Spring Boot 3.5.x parent. Dependencies:
       - spring-boot-starter-web, spring-boot-starter-security, spring-boot-starter-data-jpa, spring-boot-starter-validation
       - liquibase-core, postgresql (runtime), jjwt-api + jjwt-impl + jjwt-jackson (0.12.6), java-uuid-generator (5.1.0)
       - spring-boot-starter-test + testcontainers-postgresql (test scope)
       - Java 21, enable virtual threads

    2. Create `ProsperityApplication.java` main class in `fr.kalifazzia.prosperity`.

    3. Create `application.yml`:
       - Datasource: `jdbc:postgresql://${DB_HOST:localhost}:${DB_PORT:5432}/${DB_NAME:prosperity}`
       - JPA: hibernate ddl-auto=validate, show-sql=false
       - Liquibase: enabled, changelog=db/changelog/db.changelog-master.yaml
       - Virtual threads enabled: `spring.threads.virtual.enabled=true`
       - JWT config: `app.jwt.secret=${JWT_SECRET}`, `app.jwt.access-expiry=900` (15min), `app.jwt.refresh-expiry=2592000` (30 days)
       - Server port 8080

    4. Create `application-dev.yml` with debug logging for SQL and security.

    5. Shared domain kernel:
       - `Money.java`: Value object wrapping BigDecimal with RoundingMode.HALF_EVEN, scale 4. Factory method `of(BigDecimal)`, `of(String)`. Implements `add`, `subtract`, `negate`. Immutable record or class with equals/hashCode based on value.
       - `UserId.java`: Simple value object wrapping UUID. Factory method `generate()` using JUG UUIDv7 (`UuidCreator.getTimeOrderedEpoch()`). `of(UUID)` for existing IDs.
       - `AccountId.java`: Same pattern as UserId.

    6. `BaseEntity.java`: MappedSuperclass with `@Id UUID id`, `@CreatedDate Instant createdAt`, `@LastModifiedDate Instant updatedAt`, `@Version Long version`. Enable JPA auditing in a config class.

    7. `JacksonConfig.java`: Configure ObjectMapper for Java 8 time module, fail on unknown properties false.

    8. `GlobalExceptionHandler.java`: @RestControllerAdvice handling MethodArgumentNotValidException (400), AccessDeniedException (403), generic Exception (500). Return consistent error DTO: `{ error: String, details: Map<String,String> }`.

    9. `SecurityConfig.java`:
       - BCryptPasswordEncoder bean with strength 12
       - SecurityFilterChain: disable CSRF (stateless API), enable CORS for SvelteKit origin, stateless session management
       - Headers: CSP ("default-src 'self'"), HSTS (maxAge 31536000, includeSubDomains), X-Frame-Options DENY, X-Content-Type-Options nosniff
       - Permit all on `/api/setup/**`, `/api/auth/**`, `/actuator/health`
       - All other endpoints require authentication (JWT filter will be added in Plan 03)
  </action>

    <automated>cd prosperity-api && mvn compile -q 2>&1 | tail -5</automated>
  </verify>
  <done>Maven project compiles. Shared kernel types exist (Money with HALF_EVEN, UserId/AccountId with UUIDv7). SecurityConfig has BCrypt(12), CSP, HSTS, X-Frame-Options. GlobalExceptionHandler returns consistent error format.</done>
