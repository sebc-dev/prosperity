  <name>Task 1: Scaffold SvelteKit project with Tailwind, Paraglide, BFF skeleton, and Vitest</name>
  <files>
    prosperity-web/package.json,
    prosperity-web/svelte.config.js,
    prosperity-web/vite.config.ts,
    prosperity-web/vitest.config.ts,
    prosperity-web/tsconfig.json,
    prosperity-web/src/app.html,
    prosperity-web/src/app.css,
    prosperity-web/src/app.d.ts,
    prosperity-web/src/hooks.server.ts,
    prosperity-web/src/routes/+layout.svelte,
    prosperity-web/src/lib/api/client.ts,
    prosperity-web/src/lib/stores/preferences.svelte.ts,
    prosperity-web/project.inlang/settings.json,
    prosperity-web/messages/fr.json,
    prosperity-web/messages/en.json,
    prosperity-web/Dockerfile
  </files>
  <action>
    1. Create SvelteKit project using `npm create svelte@latest prosperity-web` (skeleton template, TypeScript, ESLint, Prettier). If the CLI is interactive, manually create the standard SvelteKit skeleton files instead.

    2. Install dependencies:
       - `npm install -D @tailwindcss/vite tailwindcss` (Tailwind CSS 4 Vite plugin)
       - `npm install @inlang/paraglide-js` (Paraglide i18n)
       - `npm install -D @sveltejs/adapter-node` (for Docker deployment)
       - `npm install -D vitest @testing-library/svelte jsdom` (testing)

    3. `vite.config.ts`: Configure with tailwindcss() Vite plugin, paraglide({ project: './project.inlang', outdir: './src/lib/i18n' }), and sveltekit() plugins in that order.

    4. `vitest.config.ts`: Configure Vitest with jsdom environment, include `src/**/*.test.ts` pattern, and resolve aliases matching SvelteKit's `$lib` path.

    5. `svelte.config.js`: Use adapter-node.

    6. `app.html`: Standard SvelteKit template with FOUC prevention inline script (reads `theme` cookie, applies `dark` class to html before paint). Set `lang="fr"` default. Include `%sveltekit.head%` and `%sveltekit.body%`.

    7. `app.css`: Import Tailwind CSS 4 with `@import "tailwindcss"`. Add `@theme` block for custom colors (primary, accent). Configure dark mode via `@variant dark (&:where(.dark, .dark *))`.

    8. `app.d.ts`: Extend `App.Locals` with `accessToken?: string` and `user?: { id: string, email: string, role: string, displayName: string }`.

    9. `hooks.server.ts`: Implement BFF auth flow skeleton:
       - `tokenRefresh` handle: if no access_token but refresh_token exists, call `${API_URL}/api/auth/refresh` to get new tokens, set httpOnly cookies (access: 15min, refresh: 30 days, secure, sameSite lax).
       - `authGuard` handle: if no access_token and path is not public (/login, /setup), redirect 303 to /login.
       - If access_token exists, decode JWT payload (base64 middle segment, no verification -- verification is API's job) to populate `event.locals.user`.
       - Export `handle = sequence(tokenRefresh, authGuard)`.
       - Define `API_URL` from env var `API_URL` defaulting to `http://localhost:8080`.

    10. `src/lib/api/client.ts`: Create `apiClient(accessToken: string)` factory returning object with `get(path)`, `post(path, body)`, `put(path, body)`, `patch(path, body)`, `delete(path)`. Each method calls `fetch(API_URL + path, ...)` with Authorization Bearer header and Content-Type application/json. Return raw Response.

    11. `src/lib/stores/preferences.svelte.ts`: Create PreferencesStore class using Svelte 5 runes ($state for theme, locale, defaultCurrency; $derived for resolvedTheme). Include `setTheme()` method that toggles `dark` class on documentElement. Export singleton instance.

    12. Paraglide setup:
        - `project.inlang/settings.json`: sourceLanguageTag "fr", languageTags ["fr", "en"], modules for paraglide.
        - `messages/fr.json`: Core UI strings in French (login, setup, settings, accounts, common actions, session_expired).
        - `messages/en.json`: Same keys in English.

    13. `src/routes/+layout.svelte`: Root layout -- minimal for now, just `<slot />` with basic html structure. Apply Tailwind base styles. Import app.css.

    14. `Dockerfile`: Multi-stage. Stage 1: Node 22 alpine, npm ci, npm run build. Stage 2: Node 22 alpine slim, copy build output, expose 3000, run with `node build`.
  </action>
  <verify>
    <automated>cd prosperity-web && npm install && npm run check 2>&1 | tail -10</automated>
  </verify>
  <done>SvelteKit project builds. Tailwind CSS applies styles. Paraglide compiles FR/EN messages. BFF hooks skeleton ready. API client factory exists. Vitest config exists. Dockerfile ready.</done>
</task>

<task type="auto">
  <name>Task 2: Create Docker Compose, Caddy config, CI workflow, and .env setup</name>
  <files>
    docker-compose.yml,

    Caddyfile,
    .env.example,

  <action>
    1. `docker-compose.yml` (production-ready):
       - `db` service: postgres:16-alpine, env vars from .env (POSTGRES_DB=prosperity, POSTGRES_USER=prosperity, POSTGRES_PASSWORD=${DB_PASSWORD}), volume pgdata, healthcheck with pg_isready, no exposed ports (internal only).
       - `api` service: build from ./prosperity-api, env vars for datasource (jdbc:postgresql://db:5432/prosperity), JWT_SECRET from .env, SPRING_THREADS_VIRTUAL_ENABLED=true. depends_on db (condition: service_healthy). No exposed ports (Caddy proxies).
       - `web` service: build from ./prosperity-web, env vars API_URL=http://api:8080, ORIGIN=${APP_ORIGIN}. depends_on api. No exposed ports (Caddy proxies).
       - Volume: pgdata

    2. `docker-compose.dev.yml` (development overlay):
       - `db`: expose ports 5432:5432
       - `api`: expose ports 8080:8080, mount source for hot reload (optional, Spring DevTools)
       - `web`: expose ports 3000:3000

    3. `Caddyfile`: Reverse proxy config for production. `{$APP_DOMAIN:prosperity.localhost}` block. Route `/api/*` to `api:8080`, everything else to `web:3000`. Enable auto-HTTPS. Add security headers (Strict-Transport-Security, X-Content-Type-Options, X-Frame-Options, Referrer-Policy, Permissions-Policy).

    4. `.env.example`: Template with all required env vars:
       ```
       DB_PASSWORD=changeme
       JWT_SECRET=changeme-generate-a-256-bit-secret
       APP_ORIGIN=https://prosperity.example.com
       APP_DOMAIN=prosperity.example.com
       ```

    5. `.github/workflows/ci.yml`:
       - Trigger: push to main, pull_request to main
       - Jobs:
         a. `backend`: Ubuntu latest, Java 21 (temurin), PostgreSQL 16 service container.
            Steps: checkout, setup-java, maven cache, `mvn verify -q` (compile + test + package).
         b. `frontend`: Ubuntu latest, Node 22.
            Steps: checkout, setup-node, npm ci in prosperity-web, `npm run check` (svelte-check), `npm run lint`, `npm run build`.
       - Both jobs run in parallel.

    6. `.gitignore`: Standard entries for Java (target/, *.class, *.jar), Node (node_modules/, .svelte-kit/, build/), IDE (.idea/, .vscode/), env (.env, !.env.example), Docker (pgdata/), OS (.DS_Store).
  </action>
