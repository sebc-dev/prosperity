  <behavior>
    - POST /api/auth/login with valid email+password returns 200 with { accessToken, refreshToken }
    - POST /api/auth/login with wrong password returns 401
    - POST /api/auth/login with unknown email returns 401
    - POST /api/auth/refresh with valid refresh token returns new { accessToken, refreshToken } (rotation)
    - POST /api/auth/refresh with expired/invalid refresh token returns 401
    - GET /api/setup/status returns { adminExists: true/false }
    - POST /api/setup with email+displayName+password creates ADMIN user and returns tokens (when no admin exists)
    - POST /api/setup returns 403 when admin already exists
    - Admin role can access @PreAuthorize("hasRole('ADMIN')") endpoints
    - Standard role gets 403 on admin-only endpoints
    - Password stored as bcrypt hash (12 rounds)
    - Responses include X-Content-Type-Options: nosniff header
    - Responses include X-Frame-Options: DENY header
    - Responses include Cache-Control: no-cache, no-store header (for secure endpoints)
  </behavior>
  <action>
    1. `SystemRole.java`: Enum with ADMIN, STANDARD.

    2. `User.java`: JPA entity mapped to `users` table. Fields: id (UUID, from BaseEntity or explicit), email, passwordHash, displayName, systemRole (SystemRole enum, stored as string), preferences (String/JSONB), forcePasswordChange (boolean), createdAt, updatedAt, version. Use @Enumerated(STRING) for role.

    3. `UserRepository.java`: JpaRepository with `Optional<User> findByEmail(String email)`, `boolean existsBySystemRole(SystemRole role)`, `Optional<User> findFirstByIdNot(UUID excludeId)` (to find the "other" user).

    4. `CustomUserDetailsService.java`: Implements UserDetailsService. Load User by email. Return UserDetails with role as `ROLE_ADMIN` or `ROLE_STANDARD`.

    5. `JwtService.java`:
       - `generateAccessToken(User user)`: Create JWT with claims: sub=user.id, email, role, displayName. Expiry from config (15min). Sign with HMAC-SHA256 using configured secret.
       - `generateRefreshToken()`: Random 64-char hex string.
       - `validateToken(String token)`: Parse and validate. Return Claims or throw.
       - `getUserIdFromToken(String token)`: Extract subject.
       - Use jjwt library (Jwts.builder(), Jwts.parser()).

    6. `RefreshToken.java`: Entity mapped to `refresh_tokens`. Fields: id, userId, tokenHash (bcrypt hash of token value), expiresAt, createdAt. The raw token is never stored.

    7. `RefreshTokenRepository.java`: JpaRepository with `Optional<RefreshToken> findByTokenHash(String hash)`, `void deleteByUserId(UUID userId)`.

    8. `AuthService.java`:
       - `login(LoginRequest req)`: Find user by email, verify password with BCryptPasswordEncoder, generate access+refresh tokens, store refresh token hash, return AuthResponse.
       - `refreshToken(RefreshRequest req)`: Find refresh token by hashing the provided token, verify not expired, delete old token (rotation), create new refresh token, generate new access token, return AuthResponse.
       - On login: delete any existing refresh tokens for the user (single-session for simplicity).

    9. `AuthController.java`: POST `/api/auth/login` (LoginRequest -> AuthResponse), POST `/api/auth/refresh` (RefreshRequest -> AuthResponse). Both return 200. Login returns 401 on bad credentials.

    10. `SetupService.java`:
        - `isAdminExists()`: Check `userRepository.existsBySystemRole(ADMIN)`.
        - `createAdmin(SetupRequest req)`: If admin exists, throw. Otherwise create User with ADMIN role, hash password, save, auto-login (generate tokens), return AuthResponse. Use @Transactional.

    11. `SetupController.java`: GET `/api/setup/status` returns `{ adminExists: boolean }`. POST `/api/setup` accepts SetupRequest (email, displayName, password), returns AuthResponse or 403.

    12. `JwtAuthenticationFilter.java`: Extends OncePerRequestFilter. Extract Bearer token from Authorization header. Validate with JwtService. Load UserDetails. Set SecurityContext. Skip filter for public paths.

    13. Update `SecurityConfig.java` (from Plan 01): Add JwtAuthenticationFilter before UsernamePasswordAuthenticationFilter. Configure AuthenticationManager bean. Keep existing header config and public paths.

    14. DTOs as Java records:
        - `LoginRequest(String email, String password)` with @NotBlank on both
        - `AuthResponse(String accessToken, String refreshToken)`
        - `RefreshRequest(String refreshToken)` with @NotBlank
        - `SetupRequest(String email, String displayName, String password)` with @NotBlank, @Email

    15. Tests with @SpringBootTest + Testcontainers PostgreSQL:
        - `AuthControllerTest`: Test login success (200 + tokens), login failure (401), refresh success (new tokens, old token invalidated), refresh failure (401).
        - `AuthorizationTest`: Create admin + standard users. Test admin can access admin endpoint, standard gets 403. Test unauthenticated gets 401.
        - `SecurityHeadersTest`: @SpringBootTest + MockMvc. Verify responses include X-Content-Type-Options: nosniff, X-Frame-Options: DENY, and Cache-Control headers. Test both public and authenticated endpoints.
  </action>
