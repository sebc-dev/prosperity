  <behavior>
    - POST /api/accounts with type=PERSONAL creates account with MANAGE permission for owner only
    - POST /api/accounts with type=SHARED creates account with MANAGE for owner + WRITE for other user
    - GET /api/accounts returns only accounts where current user has permissions (personal = owner only, shared = both)
    - User A cannot see User B's personal accounts in GET /api/accounts
    - Both users can see shared accounts
  </behavior>
  <action>
    1. `AccountType.java`: Enum PERSONAL, SHARED.
    2. `PermissionLevel.java`: Enum MANAGE (owner), WRITE (can edit), READ (view only).

    3. `Account.java`: JPA entity mapped to `accounts`. Fields: id (UUID), name, bankName, accountType (AccountType enum STRING), ownerId (UUID FK), currency (String, default "EUR"), initialBalance (BigDecimal), currentBalance (BigDecimal), color (String), createdAt, updatedAt, version. Constructor accepting all required fields.

    4. `AccountPermission.java`: JPA entity mapped to `account_permissions`. Fields: id (UUID), accountId (UUID), userId (UUID), permissionLevel (PermissionLevel enum STRING). Unique constraint on (accountId, userId).

    5. `AccountRepository.java`: JpaRepository. Custom query: `@Query("SELECT a FROM Account a JOIN AccountPermission p ON a.id = p.accountId WHERE p.userId = :userId") List<Account> findAllByUserId(UUID userId)`.

    6. `PermissionRepository.java`: JpaRepository. `findByAccountIdAndUserId`, `findAllByAccountId`.

    7. `AccountService.java`:
       - `createAccount(CreateAccountRequest req, UUID currentUserId)`: Create Account entity with UUIDv7 id. Save. Grant MANAGE permission to currentUserId. If SHARED: find the other user via UserRepository, grant WRITE permission. Return AccountDto.
       - `getAccountsForUser(UUID userId)`: Query accounts where user has any permission. Return List<AccountDto> with accountType included for frontend grouping.
       - AccountDto includes: id, name, bankName, accountType, currency, initialBalance, currentBalance (both same in Phase 1), color, permissionLevel.

    8. `AccountController.java`:
       - POST `/api/accounts`: @RequestBody CreateAccountRequest, extract current user from SecurityContext, call service, return 201.
       - GET `/api/accounts`: extract current user, call service, return 200 with list.
       - `CreateAccountRequest` record: name (@NotBlank, @Size max 255), bankName, accountType (@NotNull), currency (default "EUR"), initialBalance (default 0), color.

    9. Tests with @SpringBootTest + Testcontainers:
       - `AccountControllerTest`: Test create personal (only owner sees), create shared (both see), visibility isolation between users.
  </action>
  <verify>
    <automated>cd prosperity-api && mvn test -Dtest="AccountControllerTest" -q 2>&1 | tail -10</automated>
  </verify>
  <done>Account CRUD respects Personal/Shared visibility. MANAGE permission auto-granted to owner. WRITE auto-granted to other user for SHARED accounts. All tests pass.</done>
</task>

  <behavior>
    - PATCH /api/users/me/profile updates display name
    - PATCH /api/users/me/preferences updates preferences JSONB (theme, locale, currency, favoriteCategories)
    - POST /api/users/me/password changes password (requires old password match)
    - POST /api/users/me/password with wrong old password returns 400
    - POST /api/users (admin only) creates Standard user with forcePasswordChange=true
    - Standard user cannot call POST /api/users (403)
    - GET /api/categories returns default + user-created categories
    - POST /api/categories creates custom category
  </behavior>
  <action>
    1. `UserService.java`:
       - `updateProfile(UUID userId, UpdateProfileRequest req)`: Update displayName. Return UserDto.
       - `updatePreferences(UUID userId, UpdatePreferencesRequest req)`: Merge preferences JSON. Return UserDto. Store as JSONB string: `{"theme":"dark","locale":"en","defaultCurrency":"EUR","favoriteCategories":["cat-id-1"]}`.
       - `changePassword(UUID userId, ChangePasswordRequest req)`: Verify old password with BCrypt, hash new password, save. Throw if old password wrong.
       - `createUser(CreateUserRequest req)`: @PreAuthorize("hasRole('ADMIN')"). Create user with STANDARD role, forcePasswordChange=true. Hash temp password. Return UserDto.
       - `getCurrentUser(UUID userId)`: Return UserDto with preferences.
       - `listUsers()`: @PreAuthorize("hasRole('ADMIN')"). Return all users as List<UserDto>.

    2. `UserController.java`:
        - GET `/api/users/me`: Return current user profile + preferences.
        - PATCH `/api/users/me/profile`: UpdateProfileRequest (displayName @NotBlank).
        - PATCH `/api/users/me/preferences`: UpdatePreferencesRequest (theme, locale, defaultCurrency, favoriteCategories).
        - POST `/api/users/me/password`: ChangePasswordRequest (oldPassword, newPassword, confirmPassword -- validate match).
        - GET `/api/users`: Admin only. List all users.
        - POST `/api/users`: Admin only. CreateUserRequest (email @Email, displayName, password).

    3. `Category.java`: Entity mapped to `categories`. Fields: id, nameKey, icon, isDefault, createdBy (nullable UUID), createdAt, updatedAt.
    4. `CategoryRepository.java`: `findByIsDefaultTrue()`, `findByCreatedBy(UUID userId)`.
    5. `CategoryService.java`: `getCategories(UUID userId)` returns defaults + user-created. `createCategory(nameKey, icon, userId)` creates custom category.
    6. `CategoryController.java`: GET `/api/categories`, POST `/api/categories`.

    7. DTOs as Java records:
       - `UpdateProfileRequest(String displayName)` with @NotBlank
       - `UpdatePreferencesRequest(String theme, String locale, String defaultCurrency, List<String> favoriteCategories)`
       - `ChangePasswordRequest(String oldPassword, String newPassword, String confirmPassword)`
       - `CreateUserRequest(String email, String displayName, String password)` with @Email, @NotBlank
       - `UserDto(UUID id, String email, String displayName, String systemRole, Object preferences, boolean forcePasswordChange)`

    8. Tests with @SpringBootTest + Testcontainers:
       - `UserServiceTest`: Test updateProfile, updatePreferences, changePassword (success + wrong old password), createUser (admin only), password stored as bcrypt.
