  <action>
    1. **UI Component Library** (src/lib/components/ui/) -- Create reusable components with Tailwind, Linear/Vercel design language:
       - `Button.svelte`: Props: variant (primary/secondary/ghost/danger), size (sm/md/lg), disabled, loading. Tailwind focus-visible ring, transitions.
       - `Input.svelte`: Props: label, name, type, error, placeholder, required. Floating or stacked label. Error styling with red border + message.
       - `Card.svelte`: Props: padding (default p-6), class (pass-through). Subtle border, rounded-lg, optional hover state.
       - `Badge.svelte`: Props: variant (default/success/warning/info), size. Rounded-full, font-medium.
       - `Select.svelte`: Props: label, name, options, value, error. Native select with custom styling.
       - `ColorPicker.svelte`: Grid of preset colors (8-10 colors). Clickable circles with checkmark on selected. Hidden input for form submission.

    2. **App layout** `(app)/+layout.svelte`:
       - Sidebar navigation: Prosperity logo/name at top, nav links (Dashboard, Accounts, Settings), user avatar/name at bottom with logout action.
       - Main content area with proper max-width and padding.
       - Responsive: sidebar on desktop, hamburger menu on mobile.
       - Dark mode support via Tailwind dark: classes.
       - Logout action: POST to a logout form action that clears cookies and redirects to /login.

    3. `(app)/+layout.server.ts`:
       - `load`: Fetch current user from `${API_URL}/api/users/me`. Return user data to all child routes. If fetch fails (token invalid), redirect to /login.

    4. `(app)/+page.svelte`: Dashboard placeholder -- simple welcome message with user's name. "Bienvenue, {displayName}" with i18n. Link to accounts. This page will be expanded in Phase 5.
