  <action>
    **Step 1: Create Angular project (per D-14)**

    From the project root `/home/negus/dev/prosperity/`:
    ```bash
    pnpm dlx @angular/cli@21 new frontend --style=css --ssr=false --skip-git --package-manager=pnpm
    ```
    This creates the `frontend/` directory with a standalone-component Angular 21 app.

    If `@angular/cli@21` is not available, try `@angular/cli@latest`. The key is Angular 21.x.

    **Step 2: Install PrimeNG and Tailwind (per D-14)**

    From `frontend/`:
    ```bash
    pnpm add primeng@21 tailwindcss-primeui
    ```

    Tailwind CSS v4 is integrated natively by Angular CLI 21 (no separate install needed -- it's part of `@angular/cli` scaffolding). If not auto-configured, run:
    ```bash
    pnpm add -D tailwindcss @tailwindcss/postcss
    ```

    **Step 3: Configure styles.css**

    Replace `frontend/src/styles.css` content with:
    ```css
    @import "tailwindcss";
    @import "tailwindcss-primeui";
    ```
    This is the CSS-based import for Tailwind v4 (NOT JS plugin -- per Research pitfall 6).

    **Step 4: Configure PrimeNG in app.config.ts**

    Update `frontend/src/app/app.config.ts` to include PrimeNG provider:
    ```typescript
    import { ApplicationConfig, providePrimeNG } from '@angular/core';

    export const appConfig: ApplicationConfig = {
      providers: [
        providePrimeNG()
      ]
    };
    ```
    Note: If `providePrimeNG` is not in `@angular/core`, import from `primeng/config` instead.

    **Step 5: Update app.component (per D-15 -- minimal, no routing/layout/theme)**

    `app.component.ts`:
    ```typescript
    import { Component } from '@angular/core';

    @Component({
      selector: 'app-root',
      standalone: true,
      template: `<h1>Prosperity</h1>`,
    })
    export class AppComponent {
      title = 'Prosperity';
    }
    ```

    Remove the default Angular boilerplate template. Keep it minimal per D-15.

    **Step 6: Add ESLint (per D-14)**

    From `frontend/`:
    ```bash
    pnpm dlx @angular/cli@21 add @angular-eslint/schematics
    ```
    This auto-configures ESLint for Angular 21 with flat config (`eslint.config.js`).

    If the schematic doesn't work, install manually:
    ```bash
    pnpm add -D @angular-eslint/schematics @angular-eslint/builder @angular-eslint/eslint-plugin @angular-eslint/eslint-plugin-template @angular-eslint/template-parser eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
    ```

    **Step 7: Add Prettier (per D-14)**

    ```bash
    pnpm add -D prettier
    ```

    Create `frontend/.prettierrc`:
    ```json
    {
      "singleQuote": true,
      "trailingComma": "all",
      "printWidth": 100,
      "tabWidth": 2,
      "semi": true
    }
    ```

    Create `frontend/.prettierignore`:
    ```
    dist/
    node_modules/
    coverage/
    .angular/
    ```

    **Step 8: Add npm scripts to package.json**

    Ensure `frontend/package.json` has these scripts (add or update):
    ```json
    {
      "scripts": {
        "start": "ng serve",
        "dev": "ng serve",
        "build": "ng build",
        "test": "ng test",
        "lint": "ng lint",
        "format:check": "prettier --check \"src/**/*.{ts,html,css,json}\"",
        "format:fix": "prettier --write \"src/**/*.{ts,html,css,json}\""
      }
    }
    ```

    **Step 9: Run Prettier to format all generated files**

    ```bash
    cd frontend && pnpm format:fix
    ```

    **Step 10: Verify everything works**

    ```bash
    cd frontend && pnpm lint && pnpm format:check && pnpm build
    ```
