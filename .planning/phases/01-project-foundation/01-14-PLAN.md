   `<!-- JaCoCo (coverage report only, no thresholds) -->`
   to:
   `<!-- JaCoCo (coverage report + threshold enforcement per INFR-08) -->`

2. Add a third `<execution>` block inside the JaCoCo plugin's `<executions>`, after the existing `report` execution (after line 183, before the closing `</executions>` tag). The new execution:

```xml
          <execution>
            <id>check</id>
            <phase>verify</phase>
            <goals>
              <goal>check</goal>
            </goals>
            <configuration>
              <rules>
                <rule>
                  <element>BUNDLE</element>
                  <limits>
                    <limit>
                      <counter>INSTRUCTION</counter>
                      <value>COVEREDRATIO</value>
                      <minimum>0.70</minimum>
                    </limit>
                    <limit>
                      <counter>BRANCH</counter>
                      <value>COVEREDRATIO</value>
                      <minimum>0.50</minimum>
                    </limit>
                  </limits>
                </rule>
              </rules>
            </configuration>
          </execution>
```

This sets:
- 70% instruction coverage minimum (BUNDLE-level, overall project)
- 50% branch coverage minimum (reasonable for a project with JPA entities that have generated branches)

These thresholds are deliberately moderate -- they enforce meaningful coverage without blocking builds on entity boilerplate branches. They can be raised as the project matures.

Note on D-08 conflict: CONTEXT.md decision D-08 said "JaCoCo en mode reporting uniquement, pas de seuil bloquant." However, INFR-08 in REQUIREMENTS.md explicitly requires "Couverture de tests enforcee avec seuils minimum (echec build si non atteint)." The requirement takes precedence.
