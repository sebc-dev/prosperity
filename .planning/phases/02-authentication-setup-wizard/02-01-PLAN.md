**File 1: `V008__create_spring_session_tables.sql`**

Create the Flyway migration with the official Spring Session JDBC schema for PostgreSQL (from `org/springframework/session/jdbc/schema-postgresql.sql`):

```sql
-- Spring Session JDBC tables (PostgreSQL)
-- Source: org/springframework/session/jdbc/schema-postgresql.sql
-- Manages user sessions in PostgreSQL per D-06 (survives server restarts)

CREATE TABLE SPRING_SESSION (
    PRIMARY_ID CHAR(36) NOT NULL,
    SESSION_ID CHAR(36) NOT NULL,
    CREATION_TIME BIGINT NOT NULL,
    LAST_ACCESS_TIME BIGINT NOT NULL,
    MAX_INACTIVE_INTERVAL INT NOT NULL,
    EXPIRY_TIME BIGINT NOT NULL,
    PRINCIPAL_NAME VARCHAR(100),
    CONSTRAINT SPRING_SESSION_PK PRIMARY KEY (PRIMARY_ID)
);

CREATE UNIQUE INDEX SPRING_SESSION_IX1 ON SPRING_SESSION (SESSION_ID);
CREATE INDEX SPRING_SESSION_IX2 ON SPRING_SESSION (EXPIRY_TIME);
CREATE INDEX SPRING_SESSION_IX3 ON SPRING_SESSION (PRINCIPAL_NAME);

CREATE TABLE SPRING_SESSION_ATTRIBUTES (
    SESSION_PRIMARY_ID CHAR(36) NOT NULL,
    ATTRIBUTE_NAME VARCHAR(200) NOT NULL,
    ATTRIBUTE_BYTES BYTEA NOT NULL,
    CONSTRAINT SPRING_SESSION_ATTRIBUTES_PK PRIMARY KEY (SESSION_PRIMARY_ID, ATTRIBUTE_NAME),
    CONSTRAINT SPRING_SESSION_ATTRIBUTES_FK
        FOREIGN KEY (SESSION_PRIMARY_ID) REFERENCES SPRING_SESSION(PRIMARY_ID) ON DELETE CASCADE
);
```

**File 2: `application.yml`** — Add these sections to the existing file:

Under `spring:` (add after `flyway:` block):
```yaml
  session:
    jdbc:
      initialize-schema: never
      table-name: SPRING_SESSION
```

Add new top-level `server:` block:
```yaml
server:
  servlet:
    session:
      timeout: 30m
      cookie:
        http-only: true
        same-site: lax
        name: SESSION
```

The `initialize-schema: never` is critical — it prevents Spring Session from trying to create tables that Flyway already manages (Research pitfall 3). The 30-minute timeout implements D-05. The httpOnly cookie implements D-07.
