# Phase 2 — UI Design Contract

> Visual and interaction contract for Authentication & Setup Wizard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Angular — shadcn not applicable) |
| Preset | not applicable |
| Component library | PrimeNG 21.x (Aura theme via tailwindcss-primeui) |
| Icon library | PrimeIcons (bundled with PrimeNG) |
| Font | System font stack: `ui-sans-serif, system-ui, sans-serif` (Tailwind default) |

**Rationale:** PrimeNG Aura is the default CSS-only theme integrated via `tailwindcss-primeui`. No additional theme configuration needed. PrimeNG's CSS variables (`--p-*`) provide the design tokens. System font stack avoids external font loading for a self-hosted app.

---

## Spacing Scale

Declared values (multiples of 4, Tailwind utility classes):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `p-1` / `gap-1` | Icon gaps, inline padding |
| sm | 8px | `p-2` / `gap-2` | Compact element spacing, input internal padding |
| md | 16px | `p-4` / `gap-4` | Default element spacing, form field gaps |
| lg | 24px | `p-6` / `gap-6` | Card padding, section padding |
| xl | 32px | `p-8` / `gap-8` | Auth form container padding |
| 2xl | 48px | `p-12` / `gap-12` | Page-level vertical spacing |
| 3xl | 64px | `p-16` / `gap-16` | Not used in this phase |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-normal` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium leading-snug` |
| Heading | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold leading-tight` |
| Subheading | 18px | 600 (semibold) | 1.3 | `text-lg font-semibold` |

**Notes:**
- All form labels use the Label role (14px/500)
- Page titles (e.g. "Creer le compte administrateur") use Heading role
- Error messages and helper text use Body role at 14px: `text-sm font-normal`
- Weights are limited to 400, 500, and 600

## Color

PrimeNG Aura theme provides the color tokens via CSS variables. This contract uses PrimeNG's semantic color system.

| Role | CSS Variable / Value | Usage |
|------|---------------------|-------|
| Dominant (60%) | `--p-surface-0` (#ffffff) | Page background, form backgrounds |
| Secondary (30%) | `--p-surface-50` / `--p-surface-100` | Auth card background, layout header, sidebar |
| Accent (10%) | `--p-primary-color` (Aura blue) | Primary CTA buttons, focus rings, active links |
| Destructive | `--p-red-500` | Not used in this phase (no destructive actions) |
| Error text | `--p-red-500` | Validation messages, login error message |
| Success | `--p-green-500` | Password validation checkmarks (criteria met) |
| Muted text | `--p-text-muted-color` | Helper text, secondary labels |

**Accent reserved for (explicit list):**
- "Creer le compte" button on setup wizard
- "Se connecter" button on login page
- "Deconnexion" button in header (text style, not filled)
- Focus rings on input fields
- Active route indicator in sidebar (future phases)

**Dark mode:** Not in scope for Phase 2. PrimeNG Aura supports it natively; can be enabled later without contract changes.

## Password Validation UI (D-14)

Real-time feedback below the password field during setup:

| Criterion | Copy | Icon |
|-----------|------|------|
| Min length | 12 caracteres minimum | Checkmark (green) or X (muted) |
| Uppercase | 1 lettre majuscule | Checkmark (green) or X (muted) |
| Digit | 1 chiffre | Checkmark (green) or X (muted) |
| Special char | 1 caractere special | Checkmark (green) or X (muted) |

- Each criterion is a row: `pi-check` (green) when met, `pi-times` (muted) when not met
- Font: 14px, regular weight
- Color: `--p-green-500` when met, `--p-text-muted-color` when not met
- Layout: vertical stack below password field, `gap-1` (4px) between rows

## Interaction States

### Form Inputs

| State | Visual |
|-------|--------|
| Default | PrimeNG default border `--p-inputtext-border-color` |
| Focus | Accent ring via PrimeNG `--p-focus-ring-color` |
| Error | Red border `--p-red-500`, error message below in red 14px |
| Disabled | Not used in this phase |

### Buttons

| State | Visual |
|-------|--------|
| Default | Filled primary, accent color background |
| Hover | PrimeNG hover state (slightly darker) |
| Loading | Spinner icon replaces label text, button disabled |
| Disabled | Reduced opacity (0.6), cursor not-allowed |

### Loading States

| Action | Indicator |
|--------|-----------|
| Setup form submit | Button shows spinner + "Creation en cours..." |
| Login form submit | Button shows spinner + "Connexion..." |
| Initial app load (checking auth) | Full-screen centered spinner (PrimeNG `p-progressspinner`) |
| Session check on refresh | Same full-screen spinner |

## Routing & Guards

| Route | Guard | Behavior |
|-------|-------|----------|
| `/setup` | `noAdminGuard` | If admin exists, redirect to `/login` |
| `/login` | `unauthenticatedGuard` | If authenticated, redirect to `/dashboard` |
| `/dashboard` | `authGuard` | If not authenticated, redirect to `/login` |
| `/` | Redirect | Redirect to `/dashboard` (guard chain handles auth) |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Focus management | Auto-focus email field on setup/login page load |
| Keyboard navigation | Tab order: email > password > (display name on setup) > submit |
| Error announcement | `aria-live="polite"` on error message container |
| Password visibility toggle | `aria-label="Afficher le mot de passe"` / `"Masquer le mot de passe"` |
| Form labels | All inputs use `p-floatlabel` which generates proper `<label>` + `for` attribute |
| Color contrast | PrimeNG Aura meets WCAG 2.1 AA by default |
