  <action>
    Create `AccountService.java` in `com.prosperity.account` package. Use `@Service` annotation and constructor injection. All methods receive `String userEmail` as parameter (not SecurityContext — pitfall 6 from RESEARCH).

    Constructor dependencies:
    - `AccountRepository accountRepository`
    - `AccountAccessRepository accountAccessRepository`
    - `UserRepository userRepository` (from com.prosperity.auth)

    Implement these methods:

    **1. `createAccount(CreateAccountRequest request, String userEmail)`** — returns `AccountResponse`
    - Resolve user: `userRepository.findByEmail(userEmail).orElseThrow()`
    - Create Account: `new Account(request.name(), request.accountType())`
    - Save account: `accountRepository.save(account)`
    - D-04: Create AccountAccess with `AccessLevel.ADMIN` for creator
    - D-05: No auto-access for other users (nothing else to do)
    - Return `toResponse(account, AccessLevel.ADMIN)`
    - Annotate with `@Transactional`

    **2. `getAccounts(boolean includeArchived, String userEmail)`** — returns `List<AccountResponse>`
    - Resolve user
    - D-07: if `includeArchived` call `findAllAccessibleByUserIdIncludingArchived`, else `findAllAccessibleByUserId`
    - Map Object[] results: `result[0]` is Account, `result[1]` is AccessLevel
    - Return list of `toResponse(account, accessLevel)`

    **3. `getAccount(UUID accountId, String userEmail)`** — returns `AccountResponse`
    - Resolve user
    - Call `accountRepository.findByIdAndUserId(accountId, userId)`
    - If empty: check if account exists with `accountRepository.existsById(accountId)`:
      - If account exists but no access: throw `AccountAccessDeniedException` (D-02: 403)
      - If account doesn't exist: throw `AccountNotFoundException` (404)
    - Map and return

    **4. `updateAccount(UUID accountId, UpdateAccountRequest request, String userEmail)`** — returns `AccountResponse`
    - Resolve user
    - Fetch account+access via `findByIdAndUserId`
    - If no result: same 403/404 logic as getAccount
    - Check access level: if accessLevel is not at least WRITE, throw `AccountAccessDeniedException`
    - Apply non-null fields from request: if `request.name() != null` set name, if `request.accountType() != null` set type, if `request.archived() != null` set archived (D-08)
    - Set `account.setUpdatedAt(Instant.now())`
    - Save and return
    - Annotate with `@Transactional`

    **Helper method:** `private AccountResponse toResponse(Account account, AccessLevel accessLevel)`:
    ```java
    return new AccountResponse(
        account.getId(),
        account.getName(),
        account.getAccountType(),
        account.getBalance().amount(),
        account.getCurrency(),
        account.isArchived(),
        account.getCreatedAt(),
        accessLevel
    );
    ```

    **Helper method:** `private User resolveUser(String userEmail)`:
    ```java
    return userRepository.findByEmail(userEmail)
        .orElseThrow(() -> new RuntimeException("User not found: " + userEmail));
    ```

    - Check if access entry already exists: `accountAccessRepository.findByBankAccountIdAndUserId(accountId, request.userId())`

    - If not exists: create new `AccountAccess(targetUser, account, request.accessLevel())`, save
    - Need to load the Account entity for creating AccountAccess: `accountRepository.findById(accountId).orElseThrow()`
    - Return mapped response
