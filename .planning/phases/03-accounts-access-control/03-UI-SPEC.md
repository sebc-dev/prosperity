- **Focal point:** "Ajouter un compte" button — top right of content area, accent-filled

### Layout: Access Management Dialog (D-12)

```
+--------------------------------------------+
|  Gerer les acces — {Nom du compte}  [X]   |
|--------------------------------------------|
|                                            |
|  Utilisateurs avec acces :                 |
|                                            |
|  +----------------------------------------+|
|  | Utilisateur     | Niveau       | Actn  ||
|  |-----------------|--------------|-------||
|  | admin@ex.com    | [Admin   v]  | —     ||
|  | user2@ex.com    | [Lecture v]  | [x]   ||
|  +----------------------------------------+|
|                                            |
|  Ajouter un utilisateur :                  |
|  [Utilisateur      v] [Lecture v] [+]     |
|                                            |
|  [Error message area]                      |
|                                            |
|                         [Fermer]           |
+--------------------------------------------+
```

- Dialog width: `max-w-2xl` (672px) — wider than create dialog to fit table
- Inner table: simple, no sorting needed (household scale: 2-3 users)
- Current user's own ADMIN row: level select disabled, no remove button (cannot remove own admin access)
- "Ajouter un utilisateur" row: `p-select` for user (users not yet having access) + `p-select` for level + `pi-plus` icon button
- Remove button: `pi-times` icon, `severity="danger"`, `[text]="true"`
- Footer: single "Fermer" secondary button (changes are saved immediately per row action)
- Immediate save pattern: each access level change or user addition triggers an API call. No bulk "save all" — changes are atomic. Loading spinner on the individual row during save.

## Interaction States

### Table Rows

| State | Visual |
|-------|--------|
| Default | White background, normal text |
| Hover | `--p-surface-50` background |
| Archived | `text-muted-color`, "Archive" tag |

### Dialogs

| State | Visual |
|-------|--------|
| Opening | PrimeNG default fade-in animation |
| Form invalid | "Enregistrer" button disabled (opacity 0.6) |

### Sidebar Navigation

| State | Visual |
|-------|--------|
| Default link | `text-muted-color`, no background |
| Active link (`/accounts`) | Accent color text, `bg-surface-100` background, left border accent 3px |
| Hover | `bg-surface-50` background |

### Archive Toggle

| State | Visual |
|-------|--------|
| Off (default) | Toggle switch off, archived accounts hidden |
| On | Toggle switch on (accent color), archived accounts visible in table with muted styling |

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Table semantics | `p-table` renders native `<table>` with `<thead>` / `<tbody>`, column headers have `scope="col"` |
| Sort announcement | PrimeNG `p-table` native `aria-sort` on sortable columns |
| Dialog focus trap | PrimeNG `p-dialog` implements native focus trap and `role="dialog"` |
| Dialog close | Escape key closes dialog (PrimeNG default) |
| Action buttons | `aria-label` on icon-only buttons: "Modifier {accountName}", "Gerer les acces de {accountName}", "Archiver {accountName}" |
| Archive toggle | `aria-label="Afficher les comptes archives"` on toggle switch |
| Error announcement | `aria-live="polite"` on error message containers in dialogs |
| Confirmation dialog | `p-confirmdialog` uses `role="alertdialog"` with descriptive `aria-label` |
| Keyboard navigation | Tab order through table rows, action buttons accessible via keyboard |
| Empty state | Empty state region has `role="status"` so screen readers announce it |
