### PrimeNG Components Used

| Component | Where | Configuration |
|-----------|-------|---------------|
| `p-table` | Category list | Columns: Nom, Categorie parente, Type, Actions. `[sortField]` on Nom. No pagination (household scale: ~30-60 categories). |
| `p-dialog` | Create/edit category | `[modal]="true"`, `[closable]="true"`, `[draggable]="false"`, header via `[header]` binding |
| `p-button` | Primary CTAs, action buttons | `severity="primary"` for CTAs, `severity="secondary"` + `[text]="true"` for row actions |
| `p-inputtext` | Category name field | `pInputText` directive, full width |
| `p-treeselect` | Parent category selector (shared component) | Hierarchical display of parent > child. `[showClear]="true"` for optional parent. `selectionMode="single"`. Filters categories to show only root-level categories (max depth 2). |
| `p-floatlabel` | Form fields in dialogs | Floating label pattern, consistent with Phase 2/3 |
| `p-tag` | Category type badge | `severity="secondary"` for Systeme, `severity="info"` for Custom |
| `p-confirmdialog` | Delete confirmation | Triggered before delete action on custom categories |
| `p-message` | Error feedback | `severity="error"` for API errors, `severity="warn"` for 409 Conflict (category in use) |
| `p-tooltip` | Icon button tooltips | On action buttons (edit, delete) |

### Shared Component: Category Selector

Per D-10, a **reusable `CategorySelector` component** must be created for use in Phase 4 dialogs and future Phase 5/6 screens.

| Property | Value |
|----------|-------|
| Component name | `CategorySelector` |
| Location | `frontend/src/app/shared/category-selector.ts` |
| PrimeNG base | `p-treeselect` |
| Selection mode | `single` |
| Data format | `TreeNode[]` — parent nodes with `children` array |
| Display | Hierarchical: parent label shown, children indented |
| Clearable | Yes (`[showClear]="true"`) — parent is optional |
| Placeholder | "Categorie parente (optionnel)" in create dialog; "Categorie" in future transaction forms |
| Filter | `[filter]="true"` — type-to-search for quick category lookup |
| Input | `categories: TreeNode[]` (from parent component) |
| Output | `categorySelected: EventEmitter<string | null>` (UUID or null) |

- **Focal point:** "Ajouter une categorie" button — top right of content area, accent-filled
- Page heading: "Categories" (24px/600), left-aligned
- Table: full width, `[stripedRows]="true"`, sortable on Nom column
- Child categories: displayed flat in the table with parent name shown in "Parente" column (not nested/collapsible — keeps the table simple and consistent with p-table pattern from Phase 3)
- System categories: no action buttons (read-only per D-03/D-09)
- Custom categories: `pi-pencil` (edit) + `pi-trash` (delete) action buttons
- Parent column shows `—` (em dash) for root-level categories
- Type column: `p-tag` badge — "Systeme" (secondary) or "Custom" (info)

## Accessibility

| Requirement | Implementation |
|-------------|---------------|
| Table semantics | `p-table` renders native `<table>` with `<thead>` / `<tbody>`, column headers have `scope="col"` |
| Sort announcement | PrimeNG `p-table` native `aria-sort` on sortable columns |
| Dialog focus trap | PrimeNG `p-dialog` implements native focus trap and `role="dialog"` |
| Dialog close | Escape key closes dialog (PrimeNG default) |
| Action buttons | `aria-label` on icon-only buttons: "Modifier {categoryName}", "Supprimer {categoryName}" |
| Error announcement | `aria-live="polite"` on error message containers in dialogs |
| Confirmation dialog | `p-confirmdialog` uses `role="alertdialog"` with descriptive `aria-label` |
| Keyboard navigation | Tab order through table rows, action buttons accessible via keyboard |
| Empty state | Empty state region has `role="status"` so screen readers announce it |
| TreeSelect | PrimeNG `p-treeselect` provides native `role="tree"` with `aria-expanded` on expandable nodes |
| System vs custom distinction | Type badge provides visual + text distinction (not color-only) |
